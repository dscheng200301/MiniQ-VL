# 更新日志

本文档记录 MiniQ-VL 的重要变更。尚未正式发布的改动归入 `Unreleased`。

## Unreleased

### 修复

- DPO 改为标准的参考模型偏好目标，默认启用冻结参考模型。
- DPO 序列得分仅计算助手回答区域，避免提示 token 干扰偏好信号。
- DPO 支持直接加载包含 `config.json` 的合并 SFT 模型目录。
- GRPO 为同一提示执行多次独立生成，不再通过复制单次结果构造回答组。
- GRPO 使用分布式采样器分片样本，避免多卡重复训练相同数据。
- GRPO 不再为提示强制填充到最大序列长度。
- GRPO 组内优势标准化使用总体标准差，避免小组规模下产生无效值。
- LLM Judge 的 1 至 10 分结果统一归一化到 0 至 1。
- API 客户端增加线程安全限流、重试和环境变量兼容。
- SFT 的 LoRA 注入位置、DDP 包装与断点恢复逻辑得到修正。
- 训练 Shell 脚本改为从脚本位置推导项目路径，减少环境耦合。
