# MiniQ-VL 优化方案

基于 GRPO 训练后的评估结果，针对 Bad Case 进行迭代优化的方案记录。

---

## 方案一：Reward 权重动态调整

**思路**: 根据评估中暴露的薄弱维度，动态调整 Reward 各项权重

**实施**:
- 评估发现属性描述薄弱 → 提高 R4 (Attribute Coverage) 权重
- 评估发现多样性不足 → 提高 R5 (Diversity) 权重
- 评估发现幻觉严重 → 提高 R6 (Hallucination Penalty) 权重

**示例**:
```bash
# 第一轮：基础 Reward
python train_grpo.py --w1 0.3 --w2 0.5 --w3 0.2 --w4 0 --w5 0 --w6 0

# 第二轮：根据评估发现属性覆盖不足，加入 R4
python train_grpo.py --w1 0.2 --w2 0.4 --w3 0.15 --w4 0.15 --w5 0 --w6 0.1
```

**优点**: 实现简单，直接复用现有代码
**缺点**: 需要人工判断和调参，缺乏自动性

---

## 方案二：数据筛选与增强

**思路**: 根据 Bad Case 分析结果，针对性地筛选或增强训练数据

**实施**:
- 统计 Bad Case 的失败类型分布
- 对高频失败类型（如"属性缺失"），从数据集中增加该类型样本
- 对低频但严重的问题（如"幻觉"），构造对抗样本

**数据增强策略**:
| 失败类型 | 增强策略 |
|---|---|
| 属性缺失 | 增加含丰富属性描述的样本 |
| 场景缺失 | 增加场景/氛围描述样本 |
| 空间缺失 | 增加空间关系描述样本 |
| 幻觉错误 | 构造负样本（错误描述），训练模型识别 |
| 过于简略 | 增加长描述样本，提高 Length Penalty |
| 过于冗余 | 增加简洁描述样本，降低最优长度上限 |

**优点**: 从数据层面根本解决问题
**缺点**: 需要额外数据源或人工标注

---

## 方案三：Prompt 策略优化

**思路**: 改进 GRPO 训练的 Prompt 设计，引导模型生成更高质量的描述

**实施**:
- 基础 Prompt: `请详细描述这张图片。`
- 增强 Prompt: `请详细描述这张图片，包括：1) 主要物体及其颜色、形状、材质 2) 物体之间的空间关系 3) 整体场景和氛围`
- 分层 Prompt: 根据图像类型（人物/风景/物品）使用不同的 Prompt 模板

**Prompt 模板示例**:
```
# 人物类
请详细描述这张图片中的人物，包括外貌、穿着、动作和表情，以及周围的环境。

# 风景类
请详细描述这张风景图片，包括地形、植被、天气、光线和整体氛围。

# 物品类
请详细描述这张图片中的物品，包括颜色、形状、材质、大小和摆放位置。
```

**优点**: 无需修改模型，成本最低
**缺点**: 效果有限，无法解决模型本身的能力不足

## 方案四：迭代式 GRPO（当前采用）

**思路**: 多轮 GRPO 训练，每轮根据评估结果调整 Reward 设计和训练策略

**迭代流程**:
```
Round 1: 基础 Reward (R1+R2+R3) → 评估 → 分析 Bad Case
    ↓
Round 2: 加入 R4 (属性覆盖) + R6 (幻觉惩罚) → 评估 → 对比
    ↓
Round 3: 调整权重 + 数据增强 → 评估 → 对比
    ↓
...直到指标收敛或满足要求
```

**每轮迭代步骤**:
1. 用当前模型进行 GRPO 训练
2. 用 `eval_all.py` 评估，生成 Bad Case 报告
3. 分析 Bad Case，确定主要失败类型
4. 根据分析结果调整下一轮的 Reward 权重和训练参数
5. 用 `compare_rounds` 对比多轮结果

**对比命令**:
```bash
# 对比多轮评估结果
python eval_all.py --mode all --compare_dirs eval_output/r1,eval_output/r2,eval_output/r3
```

**扩展 Reward 说明**:

| Reward | 说明 | 范围 | 默认权重 |
|---|---|---|---|
| R1 CLIPScore | 图文对齐度 | [0, 1] | 0.3 |
| R2 LLM-Judge | 描述质量评分 | [0, 1] | 0.5 |
| R3 Length Penalty | 长度合理性 | [0, 1] | 0.2 |
| R4 Attribute Coverage | 属性覆盖率 | [0, 1] | 0.0 |
| R5 Diversity | 组内多样性 | [0, 1] | 0.0 |
| R6 Hallucination Penalty | 幻觉惩罚 | [0, 1] | 0.0 |

**迭代建议**:
- Round 1: `--w1 0.3 --w2 0.5 --w3 0.2` (基础)
- Round 2: `--w1 0.2 --w2 0.4 --w3 0.15 --w4 0.15 --w6 0.1` (加入属性覆盖+幻觉惩罚)
- Round 3: 根据评估结果微调权重，如多样性不足则 `--w5 0.1`

## 方案五：课程学习

**思路**: 从简单样本到困难样本逐步训练，先学会基础描述再提升细节

**实施**:
1. 第一阶段：只用简单图像（单物体、清晰背景）训练
2. 第二阶段：加入中等复杂度图像（多物体、有遮挡）
3. 第三阶段：使用复杂图像（多物体、复杂场景、模糊边界）

**数据分级标准**:
| 级别 | 标准 | 示例 |
|---|---|---|
| 简单 | 1-2 个主体，背景简洁 | 单个物品、单人照 |
| 中等 | 3-5 个主体，有交互关系 | 多人合照、室内场景 |
| 困难 | 5+ 主体，复杂空间关系 | 街景、人群、自然风光 |

**优点**: 训练更稳定，避免早期被困难样本干扰
**缺点**: 需要人工或自动对数据分级

## 加速配置

### 自动 DDP

检测到多 GPU 时自动启动分布式训练，无需手动 `torchrun`：

```bash
# 自动检测多 GPU 并启动 DDP
python trainer/train_grpo.py --use_wandb

# 等效于手动执行:
torchrun --nproc_per_node=N trainer/train_grpo.py --use_wandb
```

原理：检测到 `torch.cuda.device_count() > 1` 时，自动以 `torch.distributed.run` 重新启动当前脚本（默认开启）。

## 评估与迭代检查清单

每轮迭代后，检查以下指标是否改善：

- [ ] Judge 总分是否提升
- [ ] CLIPScore 是否提升
- [ ] Self-BLEU 是否降低（多样性提升）
- [ ] Bad Case 数量是否减少
- [ ] 主要失败类型的占比是否下降
- [ ] 属性覆盖率是否提升
- [ ] 幻觉率是否降低
